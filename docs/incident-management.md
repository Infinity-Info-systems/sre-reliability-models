# Incident Management

## Purpose

Incident management defines how incidents are detected, triaged, resolved, reviewed, and turned into improvement actions.

## Incident Stages

- detect
- triage
- mitigate
- recover
- review
- improve
- recur analysis
- executive follow-up

## Operating Questions

- how fast are incidents detected?
- how is severity assigned?
- how are follow-ups tracked?
- how is recurrence prevented?

## Evidence To Collect

- incident timelines
- response notes
- postmortems
- action logs

## Use

Use this page to connect incident handling with the improvement backlog so issues do not disappear after recovery.
