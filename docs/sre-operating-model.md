# SRE Operating Model

## Purpose

The operating model defines how service ownership, reliability objectives, incident work, and improvement actions are managed.

## Operating Areas

- service ownership
- SLO review
- error budget review
- incident review
- improvement backlog
- reliability reporting
- change review

## Operating Questions

- who owns each service?
- how often are SLOs reviewed?
- what happens when budgets are consumed?
- how are recurring incidents handled?

## Evidence To Collect

- service ownership list
- review notes
- improvement backlog
- incident records
- recurring incidents

## Operating Outcome

The operating model should make it clear who owns reliability, how decisions get made, and how improvements move from issue to closure.
